<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> API (PATCH) | Formulario de alerta envia PATCH — interceptor injeta Bearer token |
| API response -> DOM | Dados de alerta renderizados — React escapa HTML por padrao |
| User URL -> Router | Usuario pode digitar /alertas/:id com UUID arbitrario |
| OpenStreetMap CDN -> Browser | Tiles de mapa carregados de tile.openstreetmap.org |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | PATCH /api/alertas/{id}/ | mitigate | Select com opcoes fixas no frontend (ativo/em_atendimento/resolvido); backend rejeita valores invalidos via DRF choices field validation |
| T-06-06 | Elevation of Privilege | Rota /alertas/:id | accept | ProtectedLayout ja protege todas as rotas filhas; backend retorna 401 sem token |
| T-06-07 | Tampering | UUID na URL /alertas/:id | accept | Backend valida UUID e retorna 404 se nao encontrado; queryset nao expoe dados de outros tenants |
| T-06-08 | Information Disclosure | Popup do mapa (nome usina, provedor) | accept | Admin panel interno — usuario ja tem acesso a todos os dados |
| T-06-09 | Spoofing | Tiles OpenStreetMap | accept | Tiles sao imagens publicas sem dados sensiveis; HTTPS usado por padrao |
| T-06-10 | Tampering | campo anotacoes (textarea livre) | mitigate | React escapa HTML automaticamente na renderizacao (JSX); backend armazena como TextField sem execucao |
</threat_model>
